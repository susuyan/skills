# 工作流程

## 完整处理流程

```
1. 接收 URL
   ↓
2. 识别平台类型(Weibo/Twitter/HN/Blog/GitHub)
   ↓
3. 抓取页面内容
   - 提取标题、描述、正文
   - 收集所有外部链接
   - 展开短链接
   ↓
4. 智能分析
   - 按优先级选择主链接
   - 过滤无关资源(统计/头像/广告)
   - 识别关键词和相关链接
   ↓
5. 生成 Markdown 条目
   - 选择合适的样式
   - 生成简短描述(≤120字符)
   - 组装引用链接
   ↓
6. 归档到 Daily Links
   - 自动分类到六大板块之一
   - 检查去重
   - 写入对应日期文件
```

## 快速判断流程

### 1. 平台识别
```
URL 包含              → 平台类型
─────────────────────────────────
weibo.com, m.weibo.cn → Weibo
x.com, twitter.com    → X/Twitter
reddit.com            → Reddit
news.ycombinator.com  → Hacker News
github.com            → GitHub
其他                  → 博客/文章
```

### 2. 分类决策
```
内容特征            → 目标板块
─────────────────────────────────
有"blog"/"post"     → 📘 Read This
仓库地址(github)    → 🛠️ Tools
教程/guide           → 🔧 Try This
播客/audio/podcast  → 🎧 Listen To
箴言/原则(短)       → 🪶 Remember This
其他有价值资源      → 🤓 Fav Finds
```

### 3. 样式选择
```
内容特征              → 样式选择
─────────────────────────────────
GitHub 仓库          → GitHub Repo
官方博客(有团队信息)  → 官方博客
长文章(深度内容)     → 要点式
工具名称自解释       → 图标单行
其他                → 概念式
```

## 关键操作要点

### 主链接优先级
```
1. GitHub 仓库
2. 官方文档/主页
3. 产品/工具页面
4. 技术文章
5. 被强调的链接
```

### 社交媒体帖子
- **Weibo/Twitter/Reddit**: 优先提取被分享的资源链接
- 保留原文关键信息作为上下文
- 提及分享者(可选)

### 博客文章
- 提取标题和正文
- 识别文中的外部链接
- 选择最相关的 3-5 个

### GitHub 仓库
- 仓库名作为主链接
- README 第一段作为描述
- 关联技术栈作为关键词
