# 一人公司运作技能规划

本文档规划将"一人公司运作"拆分为多个独立的 Claude Code 技能模块。

## 业务背景

**主营业务**: 移动端业务开发，提供软件服务

**典型工作流**:
```
需求咨询 → 评估报价 → 签订合同 → 开发交付 → 验收结算 → 后期维护
```

## 设计原则

- **单一职责**: 每个技能专注一个业务领域
- **可组合**: 技能之间可以配合使用
- **渐进式**: 从核心需求开始，逐步扩展
- **实用性**: 优先解决高频痛点

## P0 - 核心技能（立即需要）

### 1. `biz-estimate` - 项目估算与报价

**痛点**: 评估工时凭感觉，报价没有依据，历史数据无法复用

**功能**:
| 命令 | 说明 |
|------|------|
| `/biz-estimate new` | 创建新的项目估算 |
| `/biz-estimate breakdown` | 功能点拆解与工时估算 |
| `/biz-estimate quote` | 生成报价单 |
| `/biz-estimate history` | 查看历史估算（用于参考） |
| `/biz-estimate compare <id>` | 对比估算 vs 实际工时 |

**估算维度**:
- 功能模块拆解（登录、列表、详情、支付...）
- 平台（iOS / Android / 跨平台）
- 复杂度系数（简单 1x / 中等 1.5x / 复杂 2x）
- 风险缓冲（建议 20-30%）

**输出示例**:
```
项目估算: XX商城 App
========================
功能模块          工时(人天)  单价    小计
─────────────────────────────────────────
用户模块            5        ¥2,000   ¥10,000
  - 登录/注册       2
  - 个人中心        2
  - 设置           1
商品模块            8        ¥2,000   ¥16,000
  - 商品列表        3
  - 商品详情        3
  - 搜索/筛选       2
订单模块           10        ¥2,000   ¥20,000
  - 购物车          3
  - 下单流程        4
  - 订单管理        3
─────────────────────────────────────────
小计               23                  ¥46,000
风险缓冲 (20%)      5                  ¥10,000
─────────────────────────────────────────
总计               28                  ¥56,000
```

**数据存储**: `~/.claude/biz/estimates/`

### 2. `biz-contract` - 软件开发合同

**痛点**: 每次都要从头写合同，验收标准不清晰，付款节点容易扯皮

**功能**:
| 命令 | 说明 |
|------|------|
| `/biz-contract new <type>` | 创建合同（开发/维护/技术顾问） |
| `/biz-contract list` | 列出所有合同 |
| `/biz-contract milestone <id>` | 管理付款里程碑 |
| `/biz-contract sign <id>` | 记录签署状态 |
| `/biz-contract template` | 管理合同模板 |

**内置模板**（针对软件开发）:
- **软件开发合同** - 明确范围、交付物、验收标准、付款节点
- **技术维护合同** - 维护期、响应时间、服务范围
- **技术顾问合同** - 按时计费、保密条款
- **保密协议 (NDA)** - 双向保密

**合同关键条款**:
```
付款节点:
  - 签约: 30%（¥16,800）
  - UI确认: 20%（¥11,200）
  - 功能完成: 30%（¥16,800）
  - 验收通过: 20%（¥11,200）

验收标准:
  - 符合需求文档描述的功能
  - 主流程无阻塞性 Bug
  - 通过 App Store / 应用市场审核

维护期: 交付后 3 个月免费 Bug 修复
```

**数据存储**: `~/.claude/biz/contracts/`

### 3. `biz-client` - 客户管理

**痛点**: 客户信息散落各处，需求变更难追溯，维护期结束时间记不住

**功能**:
| 命令 | 说明 |
|------|------|
| `/biz-client add` | 添加新客户 |
| `/biz-client list` | 列出所有客户 |
| `/biz-client show <name>` | 查看客户详情 |
| `/biz-client requirement <name>` | 记录需求/变更 |
| `/biz-client timeline <name>` | 查看合作历史 |

**客户档案包含**:
- 基本信息（公司名、联系人、微信、电话）
- 业务信息（行业、App 类型、技术栈偏好）
- 需求记录（原始需求、变更历史）
- 合作历史（项目、合同、发票）
- 维护状态（维护期截止日期、续费情况）

**输出示例**:
```

合作历史:
─────────────────────────────────────────

2025-10  XX商城 App v2.0    ¥28,000  已结清
```

**移动端开发典型流程**:
```
1. 客户咨询需求    → /biz-client add + /biz-client requirement
2. 评估报价       → /biz-estimate new → /biz-estimate quote
3. 签订合同       → /biz-contract new (导入估算数据)
4. 收签约款       → /biz-invoice create (30%)
5. 开发迭代       → /biz-project milestone + /biz-project log
6. 阶段性收款     → /biz-invoice create (按里程碑)
7. 交付验收       → /biz-project deliver
8. 尾款收取       → /biz-invoice paid
9. 进入维护期     → /biz-maintain (自动从合同同步)
10. 维护续签      → /biz-maintain renew → /biz-contract new
```

## 实施建议

### 第一阶段：商务流程
优先实现从咨询到签约的流程：
- `biz-client` - 客户和需求管理
- `biz-estimate` - 估算和报价（这是核心痛点）
- `biz-contract` - 合同管理

### 第二阶段：项目执行
实现开发和交付流程：
- `biz-project` - 项目进度和工时
- `biz-invoice` - 发票和收款

### 第三阶段：财务闭环
- `biz-finance` - 完整记账
- `biz-maintain` - 维护期管理

### 后续迭代
- `biz-analytics` - 业务数据分析
- `biz-portfolio` - 作品集
- 根据实际使用反馈优化
